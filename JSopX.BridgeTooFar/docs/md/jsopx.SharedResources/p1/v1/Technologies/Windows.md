## **Overview**  

Almost all JSopX™ Open Project EXperiences _require_ `Windows` v **`10+`** initially, but not for the reasons you might think. 

1. Yes, I am a **Windows 10+** **_.NET Shop_**. However, **Asp.NET Core**, the **.NET Framework**, and our **Visual Studio** projects are specifically designed for **cross-platform** environments, including **Windows, macOS, and Linux**.

2. A lot goes on between the `start` and `finish` of a given project development, including collaboration with one or more individuals, AI agents, and/or on other platforms. 

3. It is due to these exchanges and interactions, that we have to be extremely cautious about file integrity.

4. Various **_hidden characters_**, **_encodings_**, and **_line endings_** can creep into the files, especially when working across different operating systems, and using scripting technologies like `Python`, `Powershell`, `Node`, etc. 
  
  > **These operations can introduce inconsistencies that lead to build failures, runtime errors, or unexpected behavior.**

---

### Scripting Hygiene

- All scripts run on **Windows 10+**  
- **CRLF** enforced by `normalizing` tools whenever possible.  
- **Encoding**: UTF-8 without BOM encoding enforced
- **.gitattributes**: enforce CRLF, protect binaries

---

### OS Compatibility Hygiene

- **Platform Compatibility**: JSopX™ projects' ability to work across Windows, Mac, and Linux, but the recommendation to primarily code on Windows (v 10 Pro).
- **Backward Compatibility**: Acknowledging that JSopX™ projects have worked on Windows (v 8) but recommending staying current with Windows (v 10).
